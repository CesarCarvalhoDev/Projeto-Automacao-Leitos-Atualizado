#  Sistema de Autoatendimento Hospitalar
## Transformando a experiência do paciente com tecnologia

---

##  Visão do Projeto

Este projeto foi desenvolvido com o objetivo de resolver um problema real do ambiente hospitalar: a sobrecarga da equipe de enfermagem com solicitações operacionais simples.

A solução propõe um sistema digital onde pacientes podem realizar pedidos diretamente do leito, permitindo que a equipe foque no que realmente importa: o cuidado com a saúde.

 **Impacto direto:** melhora na eficiência operacional + melhor experiência do paciente

---

##  Problema →  Solução

###  Problema atual
- Solicitações feitas manualmente  
- Tempo de espera elevado  
- Sobrecarga da equipe de enfermagem  

###  Solução proposta
- Aplicação web/mobile para solicitações  
- Notificações em tempo real  
- Priorização automática de pedidos  

---

##  Principais Funcionalidades

-  Criação de solicitações pelo paciente  
-  Notificação automática da equipe  
-  Acompanhamento em tempo real  
-  Priorização de pedidos urgentes  
-  Relatórios para gestão hospitalar  

---

##  Arquitetura do Sistema

O projeto segue boas práticas de separação de responsabilidades:

- **Backend:** API REST (Laravel / Flask)  
- **Frontend:** Interface responsiva (SPA)  
- **Banco de Dados:** Persistência estruturada  
- **Services Layer:** Regras de negócio desacopladas  

 Foco em código limpo, escalabilidade e manutenção

---

##  Fluxo da Aplicação

```text
Paciente → Cria Solicitação → Sistema Processa → Equipe é Notificada → Atendimento → Status Atualizado
```
##  Stack Tecnológica

- **Backend:** PHP (Laravel) | Python (Flask)  
- **Frontend:** React / Vue.js  
- **Banco de Dados:** MySQL / PostgreSQL  
- **Arquitetura:** REST + Service Layer
